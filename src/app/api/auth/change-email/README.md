# POST /api/auth/change-email - 换绑邮箱

**鉴权**：需要登录

## 前置步骤

1. 先调用 `/api/auth/send-code` 获取旧邮箱验证码（type=1）
2. 调用 `/api/auth/verify-email` 验证旧邮箱
3. 调用 `/api/auth/send-code` 获取新邮箱验证码（type=3）
4. 调用本接口完成换绑

## 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `newEmail` | string | 是 | 新邮箱地址 |
| `code` | string | 是 | 新邮箱的6位验证码（type=3） |

## 请求示例

```json
{
  "newEmail": "new@example.com",
  "code": "382716"
}
```

## 成功返回

```json
{
  "message": "邮箱换绑成功"
}
```

## 错误情况

| 状态码 | 说明 |
|--------|------|
| 400 | 邮箱格式错误 / 验证码无效 |
| 401 | 未登录 |
| 409 | 该邮箱已被使用 |
| 429 | 尝试次数过多，请30分钟后再试 |
| 500 | 服务器错误 |
