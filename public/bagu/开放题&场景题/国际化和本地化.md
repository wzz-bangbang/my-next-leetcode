##### 前端国际化 主要解决 5 件事：

1. 文案翻译（自己维护的多语言文案资源文件），一般按语言分文件或者按模块分文件
2. 日期/时间/数字/货币格式（本地化格式）一般用第三方库比如 Intl
3. 路由与 SEO（多语言 URL、hreflang、索引）
4. 资源加载与性能（按需加载语言包、缓存、回退）
##### 工具：i18n vs l10n

- **i18n（internationalization）**：把系统改造成“可支持多语言/多地区”的能力（抽取文案、格式化、路由、字体等）
    
- **l10n（localization）**：把具体语言/地区资源补齐（翻译、校对、区域格式、货币、时间、法律合规）

##### 语言选择与回退策略（必须有）
- 语言来源优先级（常用）：
    1. URL（`/en/...`）或域名（`en.xxx.com`）
    2. 用户设置（profile）
    3. 浏览器 `Accept-Language`
    4. 默认语言（比如 `en`）
- 回退策略：
    - `zh-HK` 找不到 → 回退 `zh` → 回退 `en`
##### 路由与 SEO（Next.js/SSR 项目重点）
 URL 策略（常见 3 种）
1. **路径前缀**：`/en/about`, `/zh-CN/about`（最常用，SEO 友好）
2. **子域名**：`en.example.com`（更像国际站）
3. **参数**：`?lang=en`（SEO 最差，通常不选）
SEO 必做
- `<link rel="alternate" hreflang="...">`    
- canonical 规范化
- sitemap 按语言输出

##### 性能：语言包怎么加载（高频追问）
小项目：全量加载也行
- 语言包很小（几十 KB），一次加载搞定
中大型：按需加载（推荐）
- 按语言 + 按页面 namespace 分包
- 路由切换时动态 import：
    - 首屏只加载当前语言的必要 namespace
    - 其他页面进入再加载
缓存
- 语言包可走静态资源缓存（hash 文件名）

##### 总结
前端做国际化会先确定语言来源优先级（URL/用户设置/浏览器/默认）和回退策略，然后把文案抽成多语言资源文件，提供统一的 `t()` 翻译函数，支持插值、复数、namespace。日期数字货币用 `Intl` 做本地化格式化。中大型项目会做语言包按需加载和缓存，并结合 lint/CI 检测缺失 key 与硬编码文案。SSR/Next.js 场景会用多语言路由并补齐 hreflang、sitemap，保证 SEO 和首屏一致性。
