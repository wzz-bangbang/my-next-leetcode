#### 讲讲node 做BFF

node 做BFF通常用来代理前端请求，可以处理文件、注入信息、做中间层鉴权等
1. 统一进行数据聚合与裁剪：从多个后端服务获取数据，聚合后返回给前端

2. 统一鉴权并注入加密信息，可以隔离敏感信息。把获取重要的加密信息放在服务端做，然后对前端的请求增加一层转发携带accessToken，可以完成更安全的身份校验不必担心数据暴露的风险

3. 处理文件或查询数据库或进行协议转换：前端不必关心中间层，BFF可以注入必要的信息或进行文件处理，然后进行转发
   优点
   解耦前后端：前端只需对接 BFF，无需关心后端服务细节 
   提升前端性能：减少请求次数，减小数据传输量 
   灵活适配多端：可以为不同端提供定制化接口 
   安全性增强：统一鉴权，隐藏内部服务 
   便于前端开发：前端团队可以主导 BFF 开发，接口更贴合需求
   缺点：
   增加一层调用：网络延迟可能略微增加
   维护成本：需要额外维护 BFF 层代码
   可能成为瓶颈：如果设计不当，BFF 可能成为性能瓶颈
   职责边界模糊：需要明确 BFF 和后端服务的职责划分

4. 数据聚合与裁剪
   BFF 的核心职责之一是从多个后端服务获取数据，聚合后返回给前端。

5. 代理前端请求，并注入加密信息，目的是隔离敏感信息。把获取重要的加密信息放在服务端做，然后对前端的请求增加一层转发携带accessToken，可以完成更安全的身份校验不必担心数据暴露的风险
