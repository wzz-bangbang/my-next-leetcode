##### 12.8懂车帝二轮，挂了，但讲真我怎么觉得还凑合呢

三种动画实现方式的性能比较：settimeout  transform  canvas

- setTimeout / setInterval JS 主线程定时器，不跟渲染同步，延迟不稳定容易丢帧，容易卡顿，尽量不用来做 UI 动画

- CSS transform / opacity 有GPU加速，执行开销低，不触发布局/重绘，帧率高且稳定，性能最好，最适合 UI 动画

- Canvas 每帧重新绘制像素，CPU（2D）/GPU（WebGL）渲染，重绘成本高，复杂动画时 CPU/GPU 很吃力。优势是最自由的“画布”，可以完成复杂的效果。适合大量图形、游戏场景、像素级处理，WebGL 是 GPU 渲染
