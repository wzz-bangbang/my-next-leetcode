#### 如何平衡重构和需求时间冲突
(1) 风险控制
- **评估重构风险**：提前评估改动对现有业务的影响
- **保留回滚方案**：必要时做 feature flag 或分支策略
 (2) 渐进式策略
- **模块化重构**：只重构最影响维护性的模块，减少一次性改动
- **分阶段迭代**：把重构拆分到多个版本逐步上线
 (3) 团队协作
- **跨角色沟通**：和产品、测试协商重构计划，保证上线不影响业务
- **代码 review + 测试覆盖**：确保重构质量
 (4) 工具与实践
- **使用自动化工具**：静态检查、单元测试、CI/CD 提高重构安全性
- **记录和文档**：对重构部分做文档说明，方便团队理解和接手

 面试加分点：可以结合自己实际经验举例，比如某个上线紧急需求中，如何渐进式重构，体现“业务优先 + 质量保障 + 团队协作”。

#### 谈谈你对“大前端”的理解
 (1) 技术上
- **多平台统一**：大前端不仅是 Web，还包括小程序、移动端、桌面端（Electron）、后台管理系统等。
- **前端全栈意识**：涉及前端框架、Node.js、服务端渲染、前端工程化、构建工具等。
- **微前端 / 组件化 / 工程化**：在大团队、多模块、多项目的环境下，强调可复用、可组合、可扩展。通过微前端架构、共享 NPM 包、组件库统一管理等方式，保证项目整体稳定性和开发效率。
(2) 工程化和流程
- **代码规范 & CI/CD**：统一 eslint、prettier、git hook，自动化部署流程，保证团队质量。
- **版本管理 / 依赖管理**：NPM 包、私有组件库、依赖升级策略。
- **测试和监控**：单元测试、集成测试、前端性能监控、日志收集等。
(3) 团队协作和知识管理
- **文档标准化**：设计规范、接口文档、组件库文档
- **团队共享和复盘机制**：技术分享会、代码 review 流程、经验沉淀

#### 如何带领一个前端团队?如何提升团队的技术能力
 (1) 团队管理
- **任务分解 & 目标设定**：按能力分配任务，设置短期/长期目标
- **沟通机制**：定期会议、即时沟通渠道、跨组协作流程
- **绩效与成长**：明确晋升路径、技术成长规划、code review 作为学习手段
 (2) 技术能力提升
- **技术分享**：定期分享最新技术、最佳实践或项目经验
- **代码规范 & 工程化**：推动统一规范、组件化、可复用库
- **培训与实践**：安排小项目练习、Pair Programming、内部 Hackathon
- **学习新技术趋势**：关注前端社区、开源项目、前沿框架（React/Next/Vue/微前端等）
 (3) 风险控制与质量保障
- **重构与迭代策略**：控制大规模重构风险，采用渐进式重构
- **测试覆盖率**：单元、集成、E2E 测试
- **性能与监控**：前端性能指标 (FCP/LCP/TTI/FID/INP) 和异常监控

#### 平时如何学习前端技术？
 (1) 系统化学习
- **专题学习**：对某一块技术（如 React、TypeScript、前端性能优化）系统学习，而不是零散碎片化
- **做学习计划**：安排周期性学习、练习和总结
 (2) 实践与落地
- **小项目练习**：练习新框架、新工具或设计模式
- **复盘总结**：在项目中尝试新技术后，做复盘文档或者分享，巩固理解
 (3) 社交学习
- **技术分享 / 内部培训**：参加团队分享会、线上/线下 meetup
- **交流讨论**：通过评论、issue 或社群讨论问题，加深理解
 (4) 利用工具和 AI
- **查阅资料**：官方文档、源码、GitHub issue
- **借助 AI 提高效率**：快速定位问题或理解概念，但核心还是自己实践和总结

#### 讲几个你觉得会越来越重要的前端技术？
1.**服务端渲染（SSR）与全栈前端技术**
- 例如 Next.js、Nuxt.js 等框架，结合 React Server Components 或 Vue 3 的 SSR，可以提升首屏加载速度和 SEO 体验。    
- 越来越多的项目希望前端不仅负责交互，也能优化性能和 SEO，因此前端对服务端渲染的理解会越来越重要。

2️. **微前端和模块化架构** 
 - 随着大型前端项目和多团队协作增多，微前端可以让多个团队独立开发、独立部署，减少耦合。
 - 技术上涉及模块联邦（Module Federation）、Web Components、iframe 沙箱隔离等，这些能力会越来越吃香。     

3️.**前端性能优化与体验监控**
  - 包括前端性能指标（FCP、LCP、TTI、INP/FID）、懒加载、代码分割、缓存策略、离线体验（PWA）。     
  - 用户体验要求越来越高，性能优化和监控能力直接影响产品质量和业务指标。

4.**AI 与前端结合**：自动化测试、智能推荐、代码生成工具

5.**TypeScript / 强类型化前端**：在大项目中保证可维护性

6.**WebAssembly**：性能敏感场景（图形、游戏、音视频处理）
