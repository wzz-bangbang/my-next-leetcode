##### 代码层面的高级能力

- 代码
  
  - 我要写的组件是否有复用性？
  
  - 状态在哪里管理更合理？
  
  - 逻辑能不能分层？分文件？
  
  - API 调用是不是应该封装？
  
  - 错误处理有统一方式吗？
  
  - 这个地方的代码能不能更稳健？边界条件处理了吗？

- 看一个需求时，你要能拆成：
  
  - 视图组件层
  
  - 状态与数据流
  
  - 逻辑与服务调用
  
  - API 接口约束
  
  - 错误处理
  
  - loading & UX
  
  - 边界情况
  
  - 性能考量
  
  - 埋点与日志

- 项目：你要能把一个项目设计得：
  
  - 目录结构清晰
  
  - 模块清晰
  
  - 业务逻辑与 UI 隔离
    
    API 与状态集中管理
  
  - 通用函数模块化

- 工程化包含：
  
  - Webpack / Rollup / Vite 原理
  
  - Tree-shaking、代码拆分
  
  - 性能优化（你已经整理过）
  
  - Monorepo、模块化设计
  
  - CI/CD
  
  - lint 格式化
  
  - 测试（至少单测）
