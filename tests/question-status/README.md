# 题目状态模块测试用例

> 测试范围：题目状态（未开始/尝试中/已完成）的获取和更新

---

## 测试环境准备

- [ ] 数据库已启动
- [ ] 测试用户已登录

---

## 状态值说明

| 值 | 含义 |
|----|------|
| 0 | 未开始 |
| 1 | 尝试中 |
| 2 | 已完成 |

---

## 1. 获取题目状态

### TC-STATUS-GET-001 获取所有状态（代码题）
| 项目 | 内容 |
|------|------|
| **前置条件** | 用户已登录，有若干进度记录 |
| **操作步骤** | 调用 `GET /api/question-status?type=code` |
| **预期结果** | 返回 `{ statusMap: { "1": 2, "3": 1 } }` |
| **检查点** | 只返回 status > 0 的记录 |

### TC-STATUS-GET-002 获取所有状态（八股题）
| 项目 | 内容 |
|------|------|
| **操作步骤** | 调用 `GET /api/question-status?type=bagu` |
| **预期结果** | 返回八股题进度 |

### TC-STATUS-GET-003 获取单个题目状态
| 项目 | 内容 |
|------|------|
| **操作步骤** | `GET /api/question-status?questionId=1&type=code` |
| **预期结果** | 返回 `{ status: 2 }` |

### TC-STATUS-GET-004 获取无进度的题目
| 项目 | 内容 |
|------|------|
| **前置条件** | 该题无进度记录 |
| **预期结果** | 返回 `{ status: 0 }` |

### TC-STATUS-GET-005 未登录获取
| 项目 | 内容 |
|------|------|
| **前置条件** | 用户未登录 |
| **预期结果** | 状态码 401 |

---

## 2. 更新题目状态

### TC-STATUS-UPDATE-001 新建进度记录
| 项目 | 内容 |
|------|------|
| **前置条件** | 该题无进度记录 |
| **操作步骤** | `POST /api/question-status` `{ questionId: 99, questionType: 1, status: 1 }` |
| **预期结果** | 成功，数据库插入新记录 |

### TC-STATUS-UPDATE-002 更新已有进度
| 项目 | 内容 |
|------|------|
| **前置条件** | 该题有进度记录（status=1） |
| **操作步骤** | 发送 status=2 |
| **预期结果** | 成功，数据库更新 |

### TC-STATUS-UPDATE-003 设置为未开始
| 项目 | 内容 |
|------|------|
| **前置条件** | 该题有进度记录 |
| **操作步骤** | 发送 status=0 |
| **预期结果** | 成功，记录可保留或删除 |

### TC-STATUS-UPDATE-004 缺少参数
| 项目 | 内容 |
|------|------|
| **测试数据** | 缺少 questionId |
| **预期结果** | 状态码 400 |

### TC-STATUS-UPDATE-005 无效状态值
| 项目 | 内容 |
|------|------|
| **测试数据** | status=99 |
| **预期结果** | 状态码 400 或按业务处理 |

### TC-STATUS-UPDATE-006 未登录更新
| 项目 | 内容 |
|------|------|
| **前置条件** | 用户未登录 |
| **预期结果** | 状态码 401 |

---

## 3. 批量导入状态

### TC-STATUS-IMPORT-001 正常批量导入
| 项目 | 内容 |
|------|------|
| **操作步骤** | `PUT /api/question-status` `{ statusList: [...], questionType: 1 }` |
| **预期结果** | 返回 `{ success: true, count: N }` |

### TC-STATUS-IMPORT-002 空列表导入
| 项目 | 内容 |
|------|------|
| **测试数据** | `statusList: []` |
| **预期结果** | 成功，count=0 |

### TC-STATUS-IMPORT-003 覆盖已有数据
| 项目 | 内容 |
|------|------|
| **前置条件** | 部分题目已有进度 |
| **预期结果** | 导入的数据覆盖旧数据 |

---

## 4. 数据隔离

### TC-STATUS-ISOLATION-001 用户间隔离
| 项目 | 内容 |
|------|------|
| **操作步骤** | 用户A完成题目1，用户B查询题目1状态 |
| **预期结果** | 用户B的状态为0（未开始） |

### TC-STATUS-ISOLATION-002 类型间隔离
| 项目 | 内容 |
|------|------|
| **操作步骤** | 完成代码题1，查询八股题1状态 |
| **预期结果** | 八股题1状态不受影响 |

---

## 5. 前端交互

### TC-STATUS-UI-001 列表页状态图标
| 项目 | 内容 |
|------|------|
| **操作步骤** | 查看题目列表 |
| **预期结果** | 根据状态显示不同图标 |

### TC-STATUS-UI-002 标记完成按钮
| 项目 | 内容 |
|------|------|
| **操作步骤** | 在详情页点击"标记已完成" |
| **预期结果** | 1. 状态更新为2<br>2. 列表页图标更新 |

### TC-STATUS-UI-003 保存代码自动更新状态
| 项目 | 内容 |
|------|------|
| **前置条件** | 题目状态为未开始 |
| **操作步骤** | 在代码编辑器保存代码 |
| **预期结果** | 状态自动变为"尝试中"(1) |

### TC-STATUS-UI-004 刷新后状态保持
| 项目 | 内容 |
|------|------|
| **操作步骤** | 完成题目后刷新页面 |
| **预期结果** | 状态保持为已完成 |
